## virtual environments
  • check what version of Python: 

which python

  • create your environment, type 

conda create -n <yourenvironmentname> python=3.7 anaconda (or miniconda)

  • see a list of all of your conda environments

conda info -e

  • activate your Conda environments

conda activate <yourenvironmentname>

  • install a specific package, type 

conda install -n <yourenvironmentname> [package]

  • deactivate the current active Conda environments

conda deactivate.

  • delete a Conda environment and all of its related packages

conda remove -n <yourenvironmentname> --all

  • list all conda modules

conda list

replace Anaconda with miniconda

  1. Uninstall Anaconda: - Open a terminal. - Run the command conda install anaconda-clean. - Then run anaconda-clean --yes to remove the configuration files. - Manually remove the Anaconda directory, usually located at ~/anaconda3 or ~/opt/anaconda3
  2. Ensure All Anaconda References Are Removed:
    • Check your system’s environment variables (especially PATH) to make sure there are no references to Anaconda.
    • For Windows, this can be done in System Properties under Environment Variables.
    • For macOS/Linux, check your .bashrc, .bash_profile, or .zshrc files.
  3. Install Miniconda:
    • Download Miniconda from the official website: Miniconda Download Page.
    • Choose the version that matches your operating system and desired Python version.
  4. Install Miniconda:
    • For macOS/Linux:
      • Open a terminal.
      • Navigate to the directory containing the downloaded file.
      • Run the command bash Miniconda3-latest-Linux-x86_64.sh (adjust the file name as needed).
      • Follow the on-screen prompts.
  5. Verify Installation:
    • Open a new terminal or command prompt.
    • Run conda list. If Miniconda is installed correctly, this command should list the installed packages.
  6. Reinstall Necessary Packages:
    • Use conda install package_name to reinstall any packages you need that were previously managed by Anaconda.

pip install

If you have a non-conda pip as your default pip but conda python is your default python (as below)

Then instead of just calling pip install <package>, you can use the module flag -m with python so that it uses the anaconda python for the installation

This installs the package to the anaconda library directory rather than to the library directory associated with (the non-anaconda) pip

EDIT: The reason this works is as follows: the command pip references a specific pip file/shortcut (which -a pip tells you which one). Similarly, the command python references a specific python file (which -a python tells you which one). For one reason or another these two commands can become unsynchronized, so that your ‘default’ pip is in a different folder than your default python, and therefore is associated with a different version of python.

In contrast, the python -m pip construction does not use the shortcut that the pip command points to. Instead, it asks python to find its version of pip and use that version to install a package.
